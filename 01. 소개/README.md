# 01. 소개

## 1.1 MySQL 소개

- MySQL은 지금은 오픈소스지만 **처음부터 오픈소스는 아니었다**.
- 역사는 1979년 스웨덴 TcX 회사의 터미널 인터페이스 라이브러리 **UNIREG**에서 시작했다.
- 1994년 UNIREG가 웹 시스템의 데이터베이스로 사용되며 **MySQL 1.0이 완성**됐고, 1996년에 일반에 공개됐다.
- 2000년 개발자 몬티와 데이밧이 **MySQL AB**를 설립하며 라이선스를 FPL(Free Public License)로 전환했다.
- 2006년 현재와 같은 **엔터프라이즈 에디션과 커뮤니티 에디션의 이중 라이선스 정책**이 확립됐다.
- MySQL은 썬마이크로시스템즈를 거쳐 오라클로 인수됐지만 라이선스 정책에 큰 변화는 없었다.
- 커뮤니티 에디션은 무료로 사용할 수 있지만 **100% 무료는 아니다**.
- 엔터프라이즈와 커뮤니티는 **소스코드가 동일**하며, 과거에는 두 에디션의 차이가 패치 배포 주기 정도였다.
- MySQL은 여전히 **오픈소스 생태계를 유지하는 대표적 DBMS**다.

## 1.2 왜 MySQL인가

- MySQL은 **웹 서비스 성장과 함께 널리 채택된 오픈소스 RDBMS**다.
- 커뮤니티 기반 개발과 광범위한 사용자층 덕분에 **노하우와 자료 접근성이 높다**.
- 높은 안정성과 실전 검증이 이뤄졌으며, **대규모 서비스에서도 충분히 활용 가능한 성숙한 DBMS**로 자리 잡았다.
- 오라클에 인수된 이후 지속적으로 기능이 개선됐고, 특히 **8.0 버전부터 엔터프라이즈급 기능이 대거 강화**됐다.
- 라이선스 비용을 고려하는 기업 환경에서도 매력적인 선택지로 자리했다.
- 사용 편의성과 다양한 플랫폼 지원으로 **개발자와 DBA 모두에게 친숙한 데이터베이스**다.
